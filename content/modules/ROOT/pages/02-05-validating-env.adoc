ワークベンチに接続したので、クラスタ内のすべての期待されるサービスが適切に反応していることを確認しましょう。

ワークベンチで：

左側のナビゲーションメニューで、次のフォルダに移動します：`parasol-insurance/lab-materials/02`

. `02-05-validating.ipynb`というノートブックを開きます。

Jupyter NotebookでCellsを実行したことがない場合は、以下の操作が必要です。

.. **Restart kernel and Run all Cells** リンクをクリックします。
+
[.bordershadow]
image::02/02-05-restart-and-run.png[]
.. **Restart** をクリック :
+
[.bordershadow]
image::02/02-05-restart-kernel.png[]
+
. これらのセルを実行すると、ラボで必要なすべてのサービスが応答していることが確認できます。

出力は以下のようになります。

[source,console]
----
成功: Minioはminio.ic-shared-minio.svc.cluster.local:9000で到達可能です
成功: Giteaはgitea.gitea.svc.cluster.local:3000で到達可能です
成功: Postgres Databaseはclaimdb.ic-shared-db.svc.cluster.local:5432で到達可能です
成功: LLMサービスはllm.ic-shared-llm.svc.cluster.local:8000で到達可能です
成功: LLMサービス-FlanT5はllm-flant5.ic-shared-llm.svc.cluster.local:3000で到達可能です
成功: ModelMeshはmodelmesh-serving.ic-shared-img-det.svc.cluster.local:8033で到達可能です
成功: MilvusベクトルDBはvectordb-milvus.ic-shared-milvus.svc.cluster.local:19530で到達可能です
----

このノートブックの出力が疑わしい場合は、 {ic} の開催者に連絡してください。
