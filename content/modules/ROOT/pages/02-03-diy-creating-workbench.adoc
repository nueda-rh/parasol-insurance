= ワークベンチの作成
include::_attributes.adoc[]

重要: 事前に作成されたワークベンチを使用した場合は、このセクションを実行する必要はありません。ワークベンチの作成方法に興味がある場合のみ実行してください。

== ワークベンチの起動

* データ接続とパイプラインサーバーが完全に作成されたら、ワークベンチを作成します。
+
[.bordershadow]
image::02/02-03-create-wb.png[]
* 以下の特性を持つようにしてください。
** 名前を指定します。例えば、`My Workbench`
** イメージの選択: `CUSTOM - {ic} Workbench`
** コンテナサイズ: `Standard`
** アクセラレータ: `None`
* 以下のようになります。
+
[.bordershadow]
image::02/02-02-launch-workbench-01.png[]
注：その他のワークベンチ設定（ストレージなど）を変更する必要はありません。
* ワークベンチが完全に起動するまで待ちます。
* 起動したら、**「開く」**リンクをクリックして接続します。
+
[.bordershadow]
image::02/02-03-open-link.png[]

* 以前と同じ認証情報で認証します
* 以下の設定を受け入れるよう求められます。
+
[.bordershadow]
image::02/02-02-accept.png[]

* *Allow selected permissions*（選択した権限を許可）をクリックします
* 以下の画面が表示されるはずです。
+
[.bordershadow]
image::02/02-02-jupyter.png[]

== 共通リポジトリの Git クローン

プロトタイピング作業の一環として作成されたすべての資料にアクセスできるように、Git リポジトリの内容をクローンします。

* Git UI の使用:
** Jupyter で Git UI を開きます。
+
[.bordershadow]
image::02/git-clone-1.png[]
+
** Git リポジトリの URL を入力します。
+
[.console-input]
[source,adoc]
[subs=attributes+]
----
{git-clone-repo-url}
----
+
[.bordershadow]
image::02/git-clone-2.png[]

この時点で、プロジェクトは、これから行う作業の準備が整いました。
