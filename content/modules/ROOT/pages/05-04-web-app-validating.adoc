== 改修前システム

アプリケーションに初めて接続した際に表示されるのは、新しいバージョンです。

しかし、私たちがどこから来たのかを見るために、まずは以前のバージョンに移動して、どのようなものだったかを見てみましょう。左側の **改修前システム** メニューをクリックします。


[.bordershadow]
image::05/original-app-link.png[]

改修前システムでは、 {company-name} で使用されていた古いアプリケーションがどのようなものだったかを確認できます。各請求（'Claim 1', 'Claim 2' and 'Claim 3'のタブをクリック）をナビゲートし、インターフェースが非常に原始的で、最初に受信したメールに添付されていた画像がそのままコピーされていることを確認できます。

[.bordershadow]
image::05/original-app-1.png[]

== プロトタイプ アプリ

プロトタイプアプリは、取締役会に提示するために構築した新しいアプリケーションの最初のバージョンです。もちろん、このプロトタイプにはすべての機能が実装されているわけではありませんが、請求を支援する新しいアプリケーションの潜在能力を示すには十分です。

左側のメニューから3番目の項目「請求管理」をクリックします。

[.bordershadow]
image::05/claims-link.png[]

これで、アプリケーションで受け取ったすべての請求のリストが手に入りました。各請求に移動し（件名をクリック）、添付された画像を確認したりする、便利なインターフェースが備わっていることがわかります。

[.bordershadow]
image::05/new-app-claims.png[]

さらに、処理された請求については、この請求の概要、顧客感情分析、請求から抽出されたイベントの場所と時間、AI によって処理された事故画像を確認できます。

[.bordershadow]
image::05/new-app-1.png[]

また、請求処理をサポートするアシスタント機能もご利用いただけるようになりました。 開いている請求文書の右下にある「チャットアイコン」をクリックしてアシスタントを開きます。

[.bordershadow]
image::05/new-app-chat-icon.png[]

アシスタントが開き、請求に関する質問をすることができます。RAGノートブックで見たように、アシスタントはいくつかの文書に頼って質問に回答することができます。

[.bordershadow]
image::05/new-app-chat.png[]

これはまだ単純で荒削りなアプリケーションですが、プロトタイプであったことを思い出してください。そして、取締役会にAI強化アプリケーションの潜在能力を示すには十分であり、プロジェクトへの投資を説得するのに十分でした。
